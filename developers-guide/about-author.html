About the author of webfirmframework

Few words from the initiator

Hello webfirmframework engineers. I'm just 10+ years experienced in the field of Java programming. I recognized my talent in IT when I was four years old. It might be difficult to digest but it's a truth. I thank my grandma to support me in the childhood. I was involving in many IT projects from children hobby projects to live mission critical industry projects. I put my first step into Java programming about 10 years ago. I have worked in many Java EE projects in my career and also given consultation to many PHD students in the field of AI. So you can trust my programs and intelligence!

I don't like to reveal my name because this framework is NOT for my publicity or to show any of my talents. I don't like any of my personal relationships lead compromising this framework's features/quality. I will include only valid requests i.e various analysis stages need to be passed to consider a feature request as valid so it might take bit time to process, I like only professionalism. If you still want to know my name just address me as Webfirmframework, no worries.

There are many server side frameworks available but they are limited to an nth level modification. When I realized Java is lacking an excellent framework to develop realtime application I started doing this framework. I'm really happy that this framework is now running in billions of servers around the world and a part of them may be using it for generating reports. Three of my projects are in wffweb and two of them are in Kotlin + Java. They are running successfully without any issue. Thanks to JetBrains to invent a great language like Kotlin.
